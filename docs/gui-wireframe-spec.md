# 信号処理解析ソフト GUI 画面遷移図・ワイヤーフレーム指示書

## 1. 文書の目的

本書は、信号処理解析ソフトの GUI デザインを外部または社内の GUI デザイナーへ発注するための指示書である。

目的は以下の 2 点である。

- 画面遷移を明確にし、ユーザーがどの順序で操作するかを共通認識にすること
- 各画面で何をどこに置き、何を主操作にするかをワイヤーフレームレベルで固定すること

本書はビジュアルテイストの指定書ではない。情報設計、操作設計、画面遷移、状態変化、共通コンポーネントの定義を主目的とする。

## 2. 対象ユーザー

- 評価試験担当者
- 音響解析担当者
- 異常調査担当者
- 品質確認担当者

共通する特徴は、専門知識はあるが、日常業務では短時間で比較、抽出、深掘りを繰り返すことである。

## 3. 画面設計の基本方針

- 単一ファイル解析と大量データ比較は別画面として成立させる
- 再生位置、カーソル位置、選択区間、比較対象は画面遷移後も可能な限り保持する
- 時間領域、周波数領域、時系列トレンドなどの表示は、別業務ではなく同一文脈内の表示切替として扱う
- 大量データ一覧は表形式を主軸とし、検索、絞り込み、並べ替え、比較対象追加を優先する
- 基準ファイル、比較対象、外れ値、選択中状態は常に明確に視覚区別する
- 0 件、未選択、読み込み中、エラー時にも次の操作が分かるようにする

## 4. 必須画面一覧

本ソフトでは以下の画面を設計対象とする。

1. ホーム兼データ選択画面
2. データ一覧画面
3. 単一ファイル詳細解析画面
4. 複数ファイル比較画面
5. トレンド比較画面
6. プリセット管理画面
7. ローディング状態
8. エラー状態
9. 空状態モーダルまたは空状態表示

## 5. 主要ユーザーフロー

### フロー 1: データを開く

1. ユーザーはホーム画面を開く
2. 最近使ったデータセットを選ぶ、または新しくフォルダを指定する
3. データ一覧画面へ遷移する

保持する状態:

- 最後に使用したフォルダ
- 最近使ったフィルタ条件
- 最近使った表示プリセット

### フロー 2: 一覧から絞り込む

1. ユーザーはデータ一覧画面で検索語を入力する
2. 条件フィルタを指定する
3. 一覧件数が更新される
4. 条件に合うファイル群を確認する

保持する状態:

- 検索条件
- フィルタ条件
- 並べ替え条件
- 表示列設定

### フロー 3: 比較対象を選ぶ

1. ユーザーは一覧から 2 件以上のファイルを選ぶ
2. 選択ファイルを比較対象トレイに追加する
3. 比較画面へ遷移する

保持する状態:

- 選択ファイル群
- 一覧のフィルタ条件
- 基準ファイルの指定有無

### フロー 4: 単一ファイルを詳細解析する

1. ユーザーは一覧または比較画面から対象ファイルを選ぶ
2. 単一ファイル詳細解析画面へ遷移する
3. 波形、スペクトログラム、指標情報を確認する
4. 再生しながら同期カーソルで内容を確認する

保持する状態:

- 対象ファイル
- 直前のドメイン表示
- カーソル位置
- 選択区間

### フロー 5: 別ドメインへ切り替える

1. ユーザーは詳細画面または比較画面でドメイン切替を行う
2. 時間波形、FFT、STFT、PSD、オクターブ、包絡、トレンドのいずれかへ切り替える
3. 同じ対象と解析条件のまま結果を確認する

保持する状態:

- 対象ファイルまたは比較対象群
- 基準ファイル
- 選択区間
- 解析条件

### フロー 6: 外れ値を抽出する

1. ユーザーはトレンド比較画面を開く
2. 条件を設定し、しきい値または分布から外れ値を抽出する
3. 抽出結果を一覧とチャートで確認する
4. 必要に応じて比較画面または詳細画面へ送る

保持する状態:

- 抽出条件
- 選択中の外れ値候補
- 比較トレイへの追加状況

### フロー 7: 保存済み条件を再利用する

1. ユーザーはホーム画面または各画面からプリセットを呼び出す
2. 保存済みフィルタ、比較条件、表示条件を適用する
3. 該当画面へ遷移または画面を更新する

保持する状態:

- 適用したプリセット名
- 適用後の一覧条件
- 適用後の表示モード

## 6. 画面遷移図

以下をテキスト版の画面遷移図とする。

```text
[ホーム]
  ├─ フォルダ選択/最近使ったデータ選択 → [データ一覧]
  ├─ 最近使った比較条件選択 → [複数ファイル比較]
  └─ 保存済みビュー選択 → [データ一覧] または [トレンド比較]

[データ一覧]
  ├─ 行クリック → [単一ファイル詳細解析]
  ├─ 複数選択して比較開始 → [複数ファイル比較]
  ├─ トレンド分析へ → [トレンド比較]
  ├─ プリセット管理 → [プリセット管理]
  ├─ 0件時 → [空状態表示]
  └─ 読み込み失敗時 → [エラー状態]

[単一ファイル詳細解析]
  ├─ ドメイン切替 → 同画面内で表示更新
  ├─ 比較対象に追加 → [複数ファイル比較]
  ├─ 一覧へ戻る → [データ一覧]
  ├─ 読み込み中 → [ローディング状態]
  └─ 解析失敗時 → [エラー状態]

[複数ファイル比較]
  ├─ 比較対象を個別に詳細表示 → [単一ファイル詳細解析]
  ├─ ドメイン切替 → 同画面内で表示更新
  ├─ 外れ値確認へ → [トレンド比較]
  ├─ 比較条件保存 → [プリセット管理]
  └─ 対象0件時 → [空状態表示]

[トレンド比較]
  ├─ 外れ値を詳細表示 → [単一ファイル詳細解析]
  ├─ 外れ値を比較へ追加 → [複数ファイル比較]
  ├─ プリセット保存 → [プリセット管理]
  └─ 条件該当0件時 → [空状態表示]

[プリセット管理]
  ├─ プリセット適用 → 呼び出し元画面へ戻る
  └─ 編集/複製/削除 → 同画面更新
```

## 7. 遷移時の状態保持ルール

### 7-1. 一覧から比較へ

- 選択ファイル群を保持する
- フィルタ条件と検索条件を保持する
- 一覧へ戻ったときに同じ絞り込み状態へ戻れるようにする

### 7-2. 比較から詳細へ

- 選択した対象ファイルを引き継ぐ
- 基準ファイル情報を引き継ぐ
- 可能であればカーソル位置と選択区間を引き継ぐ

### 7-3. 詳細でのドメイン切替

- 対象ファイルを保持する
- 選択区間を保持する
- 解析条件を保持する
- 表示切替は可能な限り 2 クリック以内とする

### 7-4. トレンドから比較へ

- 抽出条件を保持する
- 選択した外れ値候補を比較対象として引き継ぐ
- トレンド画面へ戻ったときに抽出結果が保持されるようにする

## 8. ワイヤーフレーム指示書

### 8-1. ホーム兼データ選択画面

#### 画面の目的

- ユーザーが最短で対象データへ到達すること
- 前回作業を再開しやすくすること

#### レイアウト指示

- 上部: アプリ名、簡易ナビゲーション、最近の作業再開導線
- 中央左: フォルダ選択、新規データセット開始
- 中央右: 最近使ったデータセット
- 下部左: 保存済みビュー
- 下部中央: 最近使った比較条件
- 下部右: 最近の解析履歴

#### 必須部品

- フォルダ選択ボタン
- 最近使ったデータカードまたは一覧
- 保存済みビュー一覧
- 最近の比較条件一覧
- 最近の解析履歴一覧

#### 主操作

- フォルダを選ぶ
- 最近の対象を開く
- 保存済みビューを適用する

#### ワイヤー上の指示

- 初回利用者向けの空状態説明を入れる
- 再訪時は最近の作業再開を最も目立たせる

### 8-2. データ一覧画面

#### 画面の目的

- 大量データから対象ファイルを探し、絞り込み、比較対象として選定すること

#### レイアウト指示

- 上部: 検索ボックス、並べ替え、表示件数、プリセット呼び出し
- 左ペイン: 条件フィルタ
- 中央: 表形式のデータ一覧
- 右ペイン: 選択中ファイルトレイまたは詳細プレビュー
- 下部: ページングまたは件数サマリー

#### 必須部品

- 検索入力
- 条件フィルタ群
- データテーブル
- 列表示切替
- 比較対象追加ボタン
- 詳細表示ボタン
- 保存済みビュー適用導線

#### 主操作

- 検索する
- 条件で絞り込む
- 並べ替える
- 比較対象へ追加する
- 単一ファイル詳細へ進む

#### ワイヤー上の指示

- 表を中心とし、比較対象追加は各行から直感的に実行できること
- フィルタ条件は常時視認可能にすること
- 右ペインは選択中トレイと詳細プレビューを切り替え可能でもよい

### 8-3. 単一ファイル詳細解析画面

#### 画面の目的

- 1 つのファイルを深く解析し、再生と図を同期させて確認すること

#### レイアウト指示

- 上部: 再生コントロール、ドメイン切替、ズーム関連操作
- 左ペイン: ファイル情報、チャネル情報、主要指標
- 中央: 共有時間軸付きの可視化領域
- 右ペインまたは下部: ピーク一覧、注記、追加情報

#### 中央可視化領域に含めるもの

- 時間波形
- スペクトログラム
- 周波数プロファイル
- 共通タイムルーラー
- 再生カーソル
- 選択区間表示

#### 必須部品

- 再生バー
- ドメイン切替
- タイムルーラー
- メトリクスカード
- ピーク一覧
- 比較対象追加ボタン

#### 主操作

- 再生する
- クリックシークする
- ズームする
- パンする
- 範囲選択する
- 別ドメインへ切り替える
- 比較対象へ追加する

#### ワイヤー上の指示

- 再生とカーソル同期を最重要操作として扱う
- 選択区間は別ドメインへ展開する起点になるため明確に見せる
- ピーク一覧や注記は折りたたみ可能な常設領域を推奨する

### 8-4. 複数ファイル比較画面

#### 画面の目的

- 複数ファイルを同一条件で比較し、差異を短時間で把握すること

#### レイアウト指示

- 上部: 比較条件バー、表示モード切替、基準ファイル表示
- 左ペインまたは上部トレイ: 比較対象一覧
- 中央: 同期表示エリア
- 右ペインまたは下部: 差分要約、比較テーブル

#### 必須表示モード

- 並列表示
- 重ね表示
- 基準ファイル固定表示

#### 必須部品

- 比較対象チップまたはリスト
- 基準ファイルラベル
- 表示モード切替
- ドメイン切替
- 差分要約パネル
- 指標比較テーブル

#### 主操作

- 比較対象を追加する
- 比較対象を外す
- 基準ファイルを指定する
- 並列表示と重ね表示を切り替える
- 詳細解析へ降りる

#### ワイヤー上の指示

- 基準ファイルは常に固定位置で表示し、他対象と見分けやすくする
- 比較対象の増減が視覚的に分かるようにする
- 横並び主体を推奨するが、狭い画面では縦積みへの切替も検討する

### 8-5. トレンド比較画面

#### 画面の目的

- 大量データの傾向把握、外れ値抽出、条件別比較を行うこと

#### レイアウト指示

- 上部: 集計対象、指標切替、表示期間または条件サマリー
- 左ペイン: 条件フィルタ
- 中央上: トレンドチャート
- 中央下: 抽出結果一覧
- 右ペイン: 外れ値詳細または比較追加トレイ

#### 必須部品

- 条件フィルタ
- 指標選択
- トレンドチャート
- 抽出結果テーブル
- 外れ値詳細パネル
- 比較対象追加ボタン

#### 主操作

- 条件で絞り込む
- 指標を切り替える
- 外れ値を選ぶ
- 詳細解析へ進む
- 比較対象へ追加する

#### ワイヤー上の指示

- 俯瞰して見つける操作と、見つけた対象を深掘りする操作を 1 画面内でつなぐ
- 外れ値詳細はモーダルより右ペイン常設を推奨する

### 8-6. プリセット管理画面

#### 画面の目的

- 保存済みフィルタ、比較条件、表示条件を再利用可能にすること

#### レイアウト指示

- 左: プリセット種別の切替
- 中央: プリセット一覧
- 右: 選択中プリセットの詳細
- 上部: 新規作成、複製、削除、既定化

#### 必須部品

- 種別タブまたは種別リスト
- プリセット一覧
- 詳細フォーム
- 適用ボタン
- 複製ボタン
- 削除ボタン

#### 主操作

- プリセットを作成する
- 編集する
- 適用する
- 複製する
- 削除する

## 9. 状態別ワイヤーフレーム指示

以下の状態は主要画面ごとに描き分けること。

### 9-1. 初期状態

- 何も選んでいないが、次の行動が分かること

### 9-2. 0 件状態

- 条件に一致する結果がないことを示す
- 条件解除または条件見直しの導線を表示する

### 9-3. 未選択状態

- 比較対象や対象ファイルが未選択であることを示す
- 選択の方法を明示する

### 9-4. 読み込み中状態

- 処理中であることが分かること
- 長時間処理では進捗または段階表示を入れること

### 9-5. エラー状態

- 失敗理由を短く示すこと
- 再試行方法または戻り先を示すこと

- 解析条件や比較条件は常に参照できる位置に置く

## 12. 受け入れ基準

以下を満たすことを受け入れ条件とする。

1. 単一ファイル詳細、複数ファイル比較、トレンド比較の 3 系統が独立して成立していること
2. 一覧から抽出し、比較し、詳細確認するまでの導線が切れていないこと
3. 再生同期、カーソル同期、選択区間の扱いが各画面で矛盾していないこと
4. ドメイン切替が少ない操作数で成立していること
5. 基準ファイルと比較対象の区別が常に明確であること
6. 0 件、未選択、ローディング、エラー時の状態設計が含まれていること
7. 保存済み条件とプリセット再利用の設計が含まれていること

## 13. 実装チームへの補足

本書はデザイナー向け指示書であり、実装仕様書ではない。ただし、以下は実装側でも前提とする。

- 単一ファイル詳細解析画面は現行の [docs/architecture.md](architecture.md) および既存 UI 実装を拡張する方向で考える
- 複数ファイル比較画面とトレンド比較画面は、新しい画面体系として設計する
- ドメイン切替は画面遷移ではなく、同一文脈での表示切替として実装する
