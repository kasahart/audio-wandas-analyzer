# カーソルと再生機能 再設計仕様書

**日付：** 2026-05-15
**対象：** 信号処理専門家による複数トラック音声比較ワークフロー

---

## Context

信号処理専門家が複数の音声トラック（処理前後・複数アルゴリズム等）を比較するために、以下の3つのユースケースを中心に再設計する。

- **UC-1 A/B比較リスニング**：トラックを切り替えても「同じ位置」から聴ける
- **UC-2 トランジェント特定**：精密なカーソル配置からの再生確認
- **UC-4 区間ループ**：ドラッグで選択した区間を繰り返し再生して比較

現状は各トラックが独立した再生状態を持ち、ドラッグ操作はトラックオフセット調整専用になっている。これをグローバルカーソルモデルに一本化し、ドラッグをループ区間選択に割り当てる。

---

## Section 1：グローバルカーソルと再生モデル

### 状態モデルの変更

```js
// 現状
let cursorNorm = null;  // null（フリー）or number（固定）

// 新仕様
let cursorNorm = 0;          // 常に number、全トラック共有、初期値 0
let playbackStartNorm = 0;   // 再生開始時の位置を記憶（新規追加）
```

### カーソル操作

| 操作 | 動作 |
|------|------|
| 波形クリック（ドラッグなし） | グローバルカーソルをその位置に移動 ＋ ループ区間を解除 |
| `←` / `→` | カーソルを現在ズームの 1px 相当ナッジ（細かい） |
| `Shift+←` / `Shift+→` | カーソルを 100ms ナッジ（粗い） |
| `Space` | 再生 / 一時停止トグル |

### 再生の引き継ぎ（UC-1）

トラック A を再生中にトラック B の ▶ を押したとき：

1. A の再生を停止
2. A を**再生開始した位置**（`playbackStartNorm`）をグローバルカーソルに書き込む
3. B をグローバルカーソル位置から再生開始（`playbackStartNorm` を更新）

どのトラックも ▶ を押せば、常にグローバルカーソル位置から再生が始まる。

### カーソル外観の変更

- `null`（フリー）概念を廃止。カーソルは常に表示（白い破線、現状維持）
- ホバー時：半透明グレーのプレビューラインを重ねて表示し、クリックで確定

## Section 2：ループ区間とドラッグ操作（UC-4、アプローチB）

### 状態モデルの追加

```js
let loopRegion = null;  // null or { start: number, end: number }（正規化グローバル時間）
```

### ドラッグ操作の再割り当て

| 操作 | 動作 |
|------|------|
| 波形上をドラッグ | ループ区間を設定（新デフォルト） |
| `Shift` + ドラッグ | トラックオフセット調整（現状の動作を Shift に移動） |

### ループ再生の動作

- ループ区間が設定されている状態で ▶ を押すと `loopRegion.start` から再生開始
- 再生カーソルが `loopRegion.end` に達したら `loopRegion.start` に自動ジャンプ
- ループ区間がない場合はファイル末尾で停止（現状と同じ）

### 視覚的表現

- ループ区間：全トラックに半透明の青いハイライト帯を重ねて表示
- 区間の両端：グリップハンドル（縦線＋三角マーカー）。グリップをドラッグするとその端点のみ移動してループ区間をリサイズする
- 区間外はやや暗くして注目範囲を視覚化

### ツールチップ

| トリガー | 表示内容 |
|---------|---------|
| 波形エリアにホバー | 「ドラッグ: ループ区間を設定」「Shift+ドラッグ: トラックの時間をずらす」 |
| Shift+ドラッグ開始時 | トラック上に「オフセット調整中 ⇄」インジケーター表示 |
| ループ区間上にホバー | 「クリックでループ解除」 |
| ループ再生中 | ツールバーに「🔁 ループ再生中」バッジ |

## Section 3：キーボードナッジと精密カーソル操作（UC-2）

### キーボードショートカット

| キー | ナッジ量 |
|------|---------|
| `←` / `→` | 現在ズームで 1px 相当（`cursorNorm += (zoomEnd - zoomStart) / canvasWidth`、ズームインするほど精度向上） |
| `Shift+←` / `Shift+→` | 100ms 固定 |
| `Space` | 再生 / 一時停止 |

### UC-2 標準操作フロー

```
1. 波形クリック → カーソルを大まかに配置（ループ区間も解除）
2. Ctrl+ホイールでズームイン（現状操作）
3. ← / → キーで精密に追い込む
4. Space で再生して確認
5. 別トラックの ▶ を押してA/B比較（UC-1）
```

### フォーカス管理

- 波形クリック時に自動的にキャンバスにフォーカスを当てる（`tabIndex="0"` を設定）
- フォーカスがない場合はホバー時に薄いアウトラインを表示してフォーカス誘導

### ツールチップ

| トリガー | 表示内容 |
|---------|---------|
| 波形エリアにキーボードフォーカス取得時 | 「← →: カーソル移動　Shift+←→: 100ms移動　Space: 再生/停止」 |
| ツールバーのカーソル時刻表示にホバー | 「← →キーで微調整できます」 |

## 実装上の注意点

1. **ドラッグ判定の変更**：`dragState` に `isShift: boolean` を追加。`mousedown` 時に `event.shiftKey` を記録し、ドラッグ種別を判定する。
2. **`cursorNorm` の null 廃止**：`drawCursorOnCanvas` 等の null チェックをすべて削除し、常に描画するよう変更する。
3. **ループ区間描画**：`paintDecimatedPoints` の後にオーバーレイとして描画する（Layer 4 として追加）。`media/comparisonWaveform.js` と `waveformRenderer.ts` の両方を更新すること。
4. **キャンバスの `tabIndex`**：`tabIndex="0"` を設定し、`keydown` イベントを各キャンバスに登録する。

## 検証方法

1. `npm run compile && npm test` でユニットテストがパスすること
2. `npm run test:e2e:vscode` で既存機能に回帰がないこと
3. 手動確認：
   - 2トラックを開き、トラックAを再生中にBの▶を押してカーソル位置から始まることを確認
   - 波形をドラッグしてループ区間（青帯）が表示されることを確認
   - ループ区間内で▶を押して繰り返し再生されることを確認
   - クリックでカーソル移動＋ループ解除されることを確認
   - ズームイン後に矢印キーで精密なカーソル移動ができることを確認
   - ツールチップが各操作で正しく表示されることを確認
