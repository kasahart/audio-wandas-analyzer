# ComparisonPanel 設計仕様書

作成日: 2026-05-07

## 概要

複数の音声ファイルを同一タイムライン上で比較するための VS Code WebView パネル。Audacity のマルチトラックビューにインスパイアされた設計。

対象ユーザーは評価試験・音響解析担当者であり、複数の計測結果を短時間で視覚的に比較・差異確認することを主目的とする。

---

## スコープ

この仕様書が対象とするのは以下のみ：

- `ComparisonPanel` の新規実装
- ディレクトリツリーへのチェックボックス・比較ボタン追加
- `extension.ts` への compare-files ハンドラ追加

単一ファイル解析（`AnalysisPanel`）、Python バックエンド、`AnalysisResult` 型定義は変更しない。

## アーキテクチャ

### 新規ファイル

```
src/panels/ComparisonPanel.ts
```

### 変更ファイル

```
src/extension.ts          compare-files メッセージハンドラ追加・複数ファイル逐次解析
src/panels/AnalysisPanel.ts   ディレクトリツリーHTMLにチェックボックス・比較ボタン追加
src/utils/audioTarget.ts  isCompareFilesMessage 型ガード追加
```

### データフロー

```
ユーザーがツリーで複数ファイルをチェック
  → 「比較」ボタンをクリック
  → WebView が compare-files メッセージ送信 { type: 'compare-files', filePaths: string[] }
  → extension.ts が受信
  → 各ファイルを逐次 runAnalysis() → AnalysisResult[]
  → ComparisonPanel.show(extensionUri, results)
  → 全データを HTML 注入して表示
```

### ComparisonPanel エントリポイント

```ts
export class ComparisonPanel {
    public static show(
        extensionUri: vscode.Uri,
        results: AnalysisResult[],   // 2〜N 件
        panel?: vscode.WebviewPanel,
    ): vscode.WebviewPanel
}
```

`AnalysisResult` は既存の `src/panels/AnalysisPanel.ts` からインポートする。型の切り出しは将来対応。

```
┌─────────────────────────────────────────────────────────────┐

├──────────────┬──────────────────────────────────────────────┤
│  （空白）    │  タイムルーラー  0:00  0:01  0:02  0:03      │
├──────────────┼──────────────────────────────────────────────┤
│ トラックヘッダ│  波形キャンバス（Canvas）                    │
│ （130px）    │                                              │
│ ・ファイル名  │  ← 全トラック共有タイムライン              │
│ ・RMS / CH   │                                              │
│ ・基準ラベル  │                                              │
│ ・M S ✕ボタン│                                              │

├──────────────┼──────────────────────────────────────────────┤
```

### ツールバー要素

| 要素 | 動作 |
|------|------|
| 縦積み / オーバーレイ切替ボタン | 表示モードをトグル |
| 波形 / スペクトログラム切替ボタン | 全トラックの表示コンテンツを統一切替 |
| ズーム `－` `＋` | 全トラック同時に時間軸を拡縮 |
| ＋ ファイル追加 | ディレクトリツリーに戻る、または追加ダイアログ（将来対応） |

### トラックヘッダー要素

| 要素 | 動作 |
|------|------|
| ファイル名（省略表示） | ホバーでフルパスをツールチップ表示 |
| RMS・チャンネル数・サンプルレート | 読み取り専用の情報表示 |
| 基準ラベル | 1 件のみ表示。別トラックのヘッダーをクリックで基準を移動 |
| `M` ボタン | そのトラックの波形を非表示（トグル）。縦積みモードでは行を折りたたむ。オーバーレイモードではそのトラックをオーバーレイキャンバスから除外する |
| `S` ボタン | 将来対応。現時点ではグレーアウト表示（クリック不可） |
| `✕` ボタン | そのトラックを比較から除外（パネルは閉じない）。基準トラックを除外した場合は残りトラックの先頭が自動的に基準になる |
| オフセット値表示 | `+0.000s` 形式。フィールドをダブルクリックでオフセット 0 にリセット。`▲▼` ボタンで ±10ms 単位微調整 |

## インタラクション

### 共有カーソル

- 波形キャンバス上でマウスを動かすと、全トラックに同じ時刻の点線カーソルを同期表示
- クリックでカーソルを固定、再クリックで解除
- カーソル位置の時刻をツールバーに表示

### ズーム／パン

| 操作 | 動作 |
|------|------|
| ツールバー `－` / `＋` | 全トラック同時ズーム |
| `Ctrl` ＋ マウスホイール | 全トラック同時ズーム |
| `Shift` ＋ マウスホイール | 全トラック連動で横スクロール（パン） |

### 時間オフセット調整（ドラッグ＋数値入力）

各トラックは独立した時間オフセット（秒）を持ち、波形の描画開始位置をずらすことができる。オーバーレイモードで時刻を合わせて重ね合わせることが主目的。

#### ヒットテスト（オーバーレイモード）

マウスダウン時、X 位置における各トラックの `envelope.max[xIndex]` から波形の Y 座標を算出し、マウスの Y 座標との距離が最小かつ閾値 20px 以内のトラックをドラッグ対象とする。閾値内に該当トラックがなければドラッグを開始しない。

#### ホバーフィードバック

マウス移動中（ドラッグ前）に最も近い波形をリアルタイムでハイライト（線を太く・明るく）し、カーソルを `ew-resize` に変更する。距離が閾値を超えている場合はハイライトなし・通常カーソル。

#### ドラッグ中

- つかんだトラックの波形のみ左右に移動
- ヘッダーのオフセット値がリアルタイムで更新
- マウス付近に「トラック名 ＋ オフセット値」のバッジを表示
- 他トラックは静止

#### 縦積みモードのドラッグ

縦積みモードでは各トラックが独立した行なので、波形エリア内をドラッグすればそのトラックが対象になる（ヒットテスト不要）。

#### 数値入力での微調整

- ヘッダーのオフセット表示フィールドに直接入力
- `▲▼` ボタンで ±10ms 単位のステップ調整
- オフセットフィールドをダブルクリックでオフセット 0 にリセット

## 状態設計

### 空状態（全トラックを ✕ で除外した場合）

「ファイルを追加してください」メッセージと「＋ ファイル追加」ボタンを中央に表示。

### 解析失敗（1 件のファイルが解析エラーの場合）

該当トラック行に「解析失敗: エラー内容」を赤文字で表示。他のトラックは正常表示を継続。パネル全体は閉じない。

## ディレクトリツリーへの変更（AnalysisPanel）

- 各ファイル行の先頭にチェックボックスを追加

- ボタンクリックで `{ type: 'compare-files', filePaths: string[] }` を postMessage
- `audioTarget.ts` に `isCompareFilesMessage` 型ガードを追加
